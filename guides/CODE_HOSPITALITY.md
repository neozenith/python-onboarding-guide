# Code Hospitality

The art of curating your code base like you are preparing for future guests like an international friend visiting your city.

https://www.youtube.com/watch?v=U3XcUrupcYg

## Document your project using cascading information

- Start with the macro view and build pathsways for self-guided finding the micro
- Start with the information center
- Show them the key public transport routes
- Show them the important landmarks
- Clue them into how to find the hidden gems in the laneways

## Be kind, generous, empathetic and visual with documentation.

- Prefer MermaidJS Diagrams in Markdown
- Use code snippets to show tangible commands and make them easy to copy paste.
- Leverage hierarchy like lists and heading levels
- Avoid walls of text
- Make use of Playwright to record videos of golden paths, then use ffmpeg to automate generating a GIF at 5x playback speed to embed in README.md

## Teach them the culture and history

- Curate a `GLOSSARY.md` of the Domain Specific Language for the context of this project.
- Curate an `ADRs.md` of all the decisions that govern the values of the project over time. 
  - Decisions are made in context. Context changes and evolves and decisions expire and get superceded.
  - "_We value A over B **at this time**, because of X and Y, in future we may choose B over A when Z is true._"
  - You design a city differently at a population of 1,000 vs a population of 1,000,000.
